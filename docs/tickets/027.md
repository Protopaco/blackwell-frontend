# [027] Edit Employees within a pay period

**Board status:** Backlog
**Part of epic:** [013] Per-Pay-Period Configuration Snapshot
**Depends on:** [024] Capture full config snapshot at pay period creation, [025] Redirect generation reads to the snapshot

## Background

Once a pay period has its own Employees snapshot ([024]), a bookkeeper needs to be able to change who's on it after creation — new hires need to be added mid-cycle, and someone added to client config after the fact shouldn't have to wait for the next pay period.

## Scope

New service/route pair (mirroring the existing `services/employee/createEmployee.ts` / `updateEmployee.ts` and `routes/v1/employee/postEmployee.ts` / `putEmployee.ts` pattern, but targeting the pay period's report workbook instead of the client's `PayrollConfig` file):

- **Add employee to pay period**: allowed at any time. Reuses the existing "generate a timesheet for an employee added after others were already generated" mechanic.
- **Remove employee from pay period**: allowed only if that employee doesn't yet have a generated timesheet for this period. Once a timesheet exists, removal is not offered — the `includeInPayroll` checkbox on the timesheet itself is the mechanism for excluding them going forward. No hard delete of the row in either case (matches the codebase's existing no-hard-delete convention for `Employee`/`Client`/`TimesheetFolder`).

## Open questions

Exact validation rules for what "doesn't yet have a generated timesheet" checks against (tab existence vs. manifest presence) should be confirmed against `generateTimesheets.ts`'s own tab-exists check before implementation.
