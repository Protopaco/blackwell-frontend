# [029] Edit Holidays within a pay period

**Board status:** Backlog
**Part of epic:** [013] Per-Pay-Period Configuration Snapshot
**Depends on:** [024] Capture full config snapshot at pay period creation, [025] Redirect generation reads to the snapshot

## Background

Holiday designation affects which cells `generateTimesheets.ts` treats as holiday hours in the timesheet layout — baked in once at generation, never recomputed afterward. Client-wide Holidays can shift in the time between a pay period's creation and its first timesheet generation; the snapshot needs to be editable during that window so the bookkeeper isn't stuck with whatever was true at creation if a holiday gets added or corrected before anyone's timesheet exists yet.

## Scope

New service/route pair targeting the pay period's `Holidays` snapshot tab, following the existing per-domain CRUD pattern:

- Editable (add/edit/remove) from pay period creation until the first timesheet is generated for that pay period.
- Locked after that — editing Holidays post-generation wouldn't affect already-generated timesheets anyway (never regenerated) and would create inconsistent layouts between employees generated before vs. after the edit within the same pay period.

Gets its own tab/edit surface on the frontend (separate epic), not folded into a read-only display.
