# [003] Unsaved-changes guard — Payroll Report

**Board status:** Backlog
**Migrated from:** old `docs/TICKETS.md` #11.3

## Background

`PayrollReportPage`'s Total Expense edits (`editedValues`) are local/uncommitted until "Save All" — currently nothing stops navigating away with unsaved edits still pending. Deliberately deferred through the whole Payroll/Allocation Report build (slices 6-10) — same status as [004].

## Scope

Add a `useBlocker`-based guard protecting the unsaved batch before "Save All" is clicked. Explicitly separate concern from "already-saved values can be corrected later, without a fresh save" — that path doesn't need protecting, only genuinely-unsaved local edits do.
