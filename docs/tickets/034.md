# [034] Holidays — sync button in pay period header

**Board status:** Backlog
**Part of epic:** [030] Per-Pay-Period Configuration UI
**Depends on:** [029] Edit Holidays within a pay period (backend)

## Background

[029]'s backend is a single one-way corrective sync (`POST .../holiday/sync`), not CRUD — there's no add/edit/remove for an individual holiday. Per [030]'s decision, Holidays doesn't get its own tab; it lives in the pay period header alongside `PayPeriodInfoCard`.

## Scope

- New card (or an addition to `PayPeriodInfoCard`) in the pay period header showing the current Holidays snapshot, read-only.
- A "Sync from Payroll Config" action, disabled once the period has a generated timesheet (same lock point as [029]'s backend).
- Triggers [029]'s sync endpoint, then refetches the snapshot.
- Confirm whether a confirmation step is wanted before syncing, since it overwrites the current snapshot (recoverable by re-syncing, but still worth a deliberate check).
