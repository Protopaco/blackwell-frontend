# [002] Update "matches pay period report" indicator so all timesheets are read the same way

**Board status:** Backlog
**Migrated from:** old `docs/TICKETS.md` #8.23, rescoped 2026-07-22

## Background

Original framing (#8.23) was narrower: an icon+tooltip on `TimesheetStatusPage` comparing live `totalHours`/`flatRateQuantity` against the payroll report's snapshot aggregates (`totalHours`/`totalFlatRate`), "warning" severity if they disagree.

**Rescoped 2026-07-22** — the real underlying problem is architectural, not just a missing UI indicator: `getTimesheetStatuses.ts` (Timesheet Status page) and `generatePayrollReport.ts` (Payroll Report generation) each independently read timesheet data their own way (both loop per-employee calling their own read path, per the old docs' note on `readTimesheetDetail` usage). Because they don't share a read path, there's a real risk of the two pages disagreeing about a timesheet's numbers for reasons beyond "the report is just a snapshot taken earlier" — i.e. an actual consistency bug, not just an expected staleness gap.

**This is mostly `blackwell-backend` work**, tracked here anyway per the decision to keep one ticket board across both repos.

## Scope

- Unify timesheet reading into a single shared service, consumed by both `getTimesheetStatuses.ts` and `generatePayrollReport.ts`, so both pages are always working from the same read logic.
- Once unified, decide whether the original mismatch-indicator UI is still needed (if reads are guaranteed consistent, the only remaining "mismatch" case is legitimate report-staleness — worth keeping as a UX signal, but the "is this a bug or just staleness" ambiguity goes away).

## Open questions

- Exact shape of the shared read service (does it live in `blackwell-backend`'s existing service layer, replacing both call sites, or wrap them)?
- Does this affect the Payroll Report page's fetch of `getPayrollReport` on the frontend at all, or is this purely a backend-internal consolidation with no frontend change beyond possibly simplifying/removing the old mismatch-indicator plan?
