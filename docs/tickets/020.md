# [020] BUG - Timesheet Status - Timesheet links should be disabled when timesheets not generated

**Board status:** Backlog
**Reported:** 2026-07-22

## Background

**This is `blackwell-frontend` work.**

On the Timesheet Status page (and the Pay Period dashboard card, which shares the same row component), each employee's row has an "Open timesheet" icon button that opens their timesheet workbook in a new tab.

The bug is in `EmployeeTimesheetStatusRow.tsx`:

```ts
const timesheetUrl = employee.timesheetFileId ? `https://docs.google.com/spreadsheets/d/${employee.timesheetFileId}/edit` : null;
```

The link is enabled based solely on whether the employee **has a timesheet workbook at all** (`timesheetFileId`, assigned once at employee creation and effectively permanent). It is not based on whether a timesheet has actually been **generated for the current pay period**. An employee's workbook can exist with no tab for the selected pay period — `employee.status === 'NotGenerated'` (see `src/models/timesheetStatusProgression.ts`) is exactly this case, driven by the backend's `deriveTimesheetStatus.ts` (`totalHours === null`).

Result: clicking the link for a `NotGenerated` employee opens their workbook, but lands on a sheet with no tab for the pay period being viewed — confusing, looks broken.

## Scope

Disable (or otherwise visually indicate) the "Open timesheet" button when `employee.status === 'NotGenerated'`, instead of gating solely on `timesheetFileId` presence. Single fix location: `EmployeeTimesheetStatusRow.tsx` — used by both the Timesheet Status page and the Pay Period dashboard card, so one fix covers both surfaces.

Not yet decided: exact disabled treatment (grayed-out icon with a tooltip explaining why vs. hidden entirely like the no-`timesheetFileId` case already does) — small UX call, worth confirming before implementing.
