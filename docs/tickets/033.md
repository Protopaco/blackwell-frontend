# [033] FundingSources tab

**Board status:** Backlog
**Part of epic:** [030] Per-Pay-Period Configuration UI
**Depends on:** [028] Edit Activities & Funding Sources within a pay period (backend)

## Background

New tab on the Pay Period page for the period's FundingSources snapshot, mirroring [032]'s approach — reuse the existing client-wide FundingSources table and Create/Edit modal components (from `FundingSourcesManagement`) rather than building new ones.

## Scope

- New tab, new page under `client/:clientId/payPeriod/:payPeriodId`, same pattern as [032].
- Reuse the existing FundingSources table + Create/Edit dialog components, wired to the pay period's FundingSource endpoints ([028]).
- FundingSource **presence** locks at first timesheet generation for the period (same gate as Activity presence, per [028] — an Activity can never structurally reference a FundingSource missing from the snapshot).
- Same add-flow question as [032]: reuse "Create" as-is, or a picker over existing client FundingSources not yet on this period.
