# [024] Capture full config snapshot at pay period creation

**Board status:** Backlog
**Part of epic:** [013] Per-Pay-Period Configuration Snapshot

## Background

Today `createPayPeriod.ts` only appends a row to the pay period registry — the payroll report workbook doesn't get created until the first "Generate Payroll Report" click (`generatePayrollReport.ts:80-85`, via `createOAuthWorkbook`), as a bare file with no pre-built tabs.

## Scope

Change pay period creation so it also creates the payroll report workbook immediately (instead of lazily at first report generation), seeded with five new tabs copied from the client's current `PayrollConfig`:

- `Employees` — active employees only, same columns as `PayrollConfig`'s `Employees` tab (including `Status`, kept for DTO reuse even though presence in the tab is what defines roster membership)
- `Activities` — same columns as `PayrollConfig`'s `Activities` tab
- `FundingSources` — same columns as `PayrollConfig`'s `FundingSources` tab
- `Holidays` — same columns as `PayrollConfig`'s `Holidays` tab
- `Settings` — same columns as `PayrollConfig`'s `Settings` tab

All five tabs get created regardless of row count (e.g. a client with zero configured FundingSources still gets an empty `FundingSources` tab) — same constraint `readPayrollConfig.ts` already has around tabs needing to exist even when empty.

`generatePayrollReport.ts`'s existing lazy-create-if-missing branch (lines 80-85) should no longer be needed for a new pay period once this ships, since the file will already exist by the time report generation runs.

## Out of scope

Redirecting any read paths to use these new tabs — that's [025]. This ticket only covers capturing the snapshot at creation time.
