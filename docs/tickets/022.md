# [022] Toast coverage — remaining and future mutation flows

**Board status:** Backlog
**Reported:** 2026-07-22

## Background

**This is `blackwell-frontend` work.**

[001] added a global toast system (`useToast()` / `ToastProvider`, `src/state/toast/`). A first pass wired it into the mutation flows that already existed at the time: Employees (create/edit), Funding Sources (create/edit/delete), Holidays (create/edit/delete), Timesheet Folders (create/edit), Activities (create/update/delete), and the `useAsyncAction`-based Pay Period actions (Create Pay Period, Close Pay Period, Save Additional Expenses, Generate Allocation Report, Save Employee Expenses, Generate Payroll Report, Generate Timesheets).

That pass was not exhaustive — it only covered success/error try-catch patterns found in one search. This ticket tracks what's left, and gives future mutation flows a place to land if they're not wired up for toasts as they're built.

## Known gaps

- `ActivitiesManagement.tsx` writes its own inline try/catch for create/update/delete instead of using `useAsyncAction`, unlike every other simple-action call site. Worth considering whether to refactor it onto `useAsyncAction` for consistency (it would have picked up toast support for free via the shared hook), separately from the toast work itself.
- Any mutation flow added after this ticket was written that doesn't call `showToast` — sweep periodically or when noticed.

## Scope

Not a single fixed list — audit and wire up remaining/new instances as found. Close this out (or spin up a fresh version of it) once toast coverage across mutation flows is considered complete enough that new gaps are the exception, not the rule.
