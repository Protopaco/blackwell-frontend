# [000] Employee Timesheet Status — refresh/reload button

**Board status:** Backlog
**Migrated from:** old `docs/TICKETS.md` #8.18

## Background

`TimesheetStatusPage` fetches `EmployeeTimesheetStatus[]`, `EmployeeExpense[]`, and (once generated) the payroll report — all on mount/key-change only. There is currently no way for a user to manually re-run those fetches without a full page reload, e.g. after a supervisor edits hours directly in the underlying Google Sheet.

## Scope

Add a refresh/reload action to `TimesheetStatusPage` that re-runs its fetches (the same ones #8.3/#8.4 already established) without navigating away or doing a hard browser reload.

## Open questions

- Icon button near the table, or a page-level control near the other actions (Generate Timesheets / Generate Payroll Report)?
- Should this also live on `PayPeriodLayout` itself (refreshing `PayPeriod` too), or stay scoped to this page's own fetches? A related, broader idea was noted in the old docs ("PayPeriod info page — refresh button") but was not added to the new board — worth clarifying whether that broader scope is part of this ticket or a separate one.
