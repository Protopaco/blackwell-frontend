# [030] Per-Pay-Period Configuration UI (Epic)

**Board status:** Ready (epic — tracked as a group, broken into tickets 031-034)
**Depends on:** [013] Per-Pay-Period Configuration Snapshot (backend epic — fully shipped)

## Background

**This is `blackwell-frontend` work.**

Backend epic [013] gave every pay period its own config snapshot (Employees, Activities, FundingSources, Holidays, Settings), with endpoints for a bulk read ([026]) and per-domain edits/sync ([027] Employees, [028] Activities & FundingSources, [029] Holidays). None of it is exposed in the frontend yet.

Resolved via requirements discussion 2026-07-24:

- **Employees**: the existing Timesheet Status tab already shows this data (already redirected to the snapshot per [025]) — no new page or fetch needed. Rename the tab to "Employees" and add a remove action, gated by [027]'s rule (no generated timesheet yet for that employee).
- **Activities** and **FundingSources**: each gets its own new tab, **reusing the existing client-wide table + Create/Edit modal components** (`ActivitiesManagement`/`FundingSourcesManagement` and their tables/dialogs) rather than building new ones — same design language, less code. Pointed at the pay period's endpoints instead of `PayrollConfig`'s.
- **Holidays**: not a tab. Folded into the pay period header area alongside `PayPeriodInfoCard` — a read-only list plus the [029] sync button.

## Scope

Broken into:

- [031] Employees tab — rename from Timesheet Status, add remove action
- [032] Activities tab — new, reusing client-wide components
- [033] FundingSources tab — new, reusing client-wide components
- [034] Holidays — sync button in the pay period header

Payroll Report and Allocation Report tabs are unaffected. Final tab bar: Employees, Activities, FundingSources, Payroll Report, Allocation Report (5 total).

Each of 031-034 can ship independently.
