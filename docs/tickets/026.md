# [026] Bulk pay-period-config read endpoint

**Board status:** Backlog
**Part of epic:** [013] Per-Pay-Period Configuration Snapshot
**Depends on:** [024] Capture full config snapshot at pay period creation

## Background

The client-wide equivalent of this already exists: `readPayrollConfig.ts` does one batched `readTabs` call across all `PayrollConfig` tabs, and `getClientSummary.ts` aggregates multiple client-level reads into a single response DTO for the client dashboard.

## Scope

Same pattern, one level down: a `PayPeriodConfig`-style model (`employees`, `activities`, `fundingSources`, `holidays` — Settings included if useful for display, though it has no edit surface) backed by one batched read across the pay period's snapshot tabs, exposed as a single `GET` route. This is what the four planned UI tabs (Employees, Activities, FundingSources, Holidays) on the Pay Period page will hydrate from — one API call instead of four.

## Out of scope

Write/edit endpoints — those are [027]/[028]/[029]. This ticket is read-only.
