# [001] Global toast/notification system

**Board status:** Backlog
**Migrated from:** old `docs/TICKETS.md` #8.19

## Background

General-purpose, cross-cutting — not specific to any one page or epic. Repeatedly deferred through Pay Period, Employee/Holiday/Funding Source/Activity Management, all of which currently show success/failure inline (per-field errors, page-level error text) rather than via a toast.

## Scope

Not yet designed. Needs a decision on:
- Library/approach (MUI `Snackbar`-based vs. something else)
- Where the trigger lives (a shared hook/context vs. each page calling a global emit function)
- Whether this replaces existing inline error patterns or supplements them (existing pages already have working inline error handling — this shouldn't be treated as a mandatory refactor of everything already built, just a new capability for cases where a toast is the better fit)

## Open questions

- Should this ticket also cover migrating any existing inline error UI to toasts, or purely add the capability for future use?
