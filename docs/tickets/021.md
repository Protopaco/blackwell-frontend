# [021] Deep-link "Open timesheet" to the current pay period's tab

**Board status:** Backlog
**Reported:** 2026-07-22

## Background

**This spans `blackwell-backend` and `blackwell-frontend`.**

Each timesheet workbook (`timesheetFileId`) contains one tab per pay period. The "Open timesheet" link (`EmployeeTimesheetStatusRow.tsx`) currently opens the workbook at its default/last-viewed tab — not the tab for the pay period the user is actually looking at. The user has to manually find the right tab after the link opens.

Google Sheets supports deep-linking to a specific tab via a `gid` (sheet ID) in the URL fragment: `.../edit#gid=<gid>`. That `gid` is per-tab and isn't currently captured or exposed anywhere in the system — `EmployeeTimesheetStatus` only carries `timesheetFileId` (the workbook-level file ID).

## Scope

- **Backend:** when a pay period's tab is created/generated in the workbook, capture its `gid` and persist it alongside the existing per-period timesheet data. Expose it through the `EmployeeTimesheetStatus` (or equivalent) response for the selected pay period.
- **Frontend:** in `EmployeeTimesheetStatusRow.tsx`, build the timesheet URL as `https://docs.google.com/spreadsheets/d/${timesheetFileId}/edit#gid=${gid}` when a `gid` is available for the selected pay period, falling back to the current behavior otherwise.
