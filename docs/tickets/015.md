# [015] Pay Period Info Refresh Button

**Board status:** Backlog
**Migrated from:** old `docs/TICKETS.md` "PayPeriod info page — refresh button" note (noted 2026-07-20)

## Background

Noted during the `includeInPayroll` refactor (moving Include/Ignore from a payroll-report switch to a read-only checkbox on the generated timesheet itself, edited directly in the sheet by a supervisor): the UI has no way to pick up changes made outside of it — e.g. a supervisor toggling the `includeInPayroll` checkbox, or entering hours, directly in Google Sheets.

This is the broader sibling of [000] (Employee Timesheet Status refresh/reload button) — [000] covers just that one table's fetch; this ticket is about the info-page level (`PayPeriodLayout`/`PayPeriodInfoCard`) so it also catches `PayPeriod` itself and any other tab's data, not just Timesheet Status.

## Scope

A refresh/reload action on the PayPeriod info page that re-runs the page-level fetches (`PayPeriod` via `PayPeriodLayout`, plus whichever child-page fetches make sense) so the UI reflects sheet-side edits without a full browser reload.

## Open questions

- Overlap with [000]: build once at the layout level (refreshing `PayPeriod` + delegating to each child page's own refetch), or keep them as two separate, narrower actions? Worth resolving together rather than building both independently.
